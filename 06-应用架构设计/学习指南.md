## 🎯 10 条学习原则落地指南

### 1. 骨架先行 — 建立整体认知地图

> 先搞懂整体结构、模块划分、核心链路，不一开始扎进零散语法/API；先有地图，再逛景点

**本模块骨架**：MVVM→MVI→Clean Architecture→组件化→插件化，架构演进之路

**行动**：打开 `知识骨架.xmind`，花 5 分钟浏览全部节点，建立模块全局印象，再按阶段逐一攻克。

---

### 2. 最小闭环 — 先跑通再深挖

> 先做一个能完整运行的最小可用版本，不求完美、不求精通；先建立体感和全局流程，再回头拆解底层原理

**本模块最小闭环项目**：
- **项目**：三模块组件化 Demo
- **技术栈**：app(壳)+home+profile+ARouter
- **闭环目标**：壳工程组装→ARouter 路由跳转→模块间服务调用

**检查点**：能否独立跑通这个闭环？如果能，说明框架理解正确；如果不能，回头补最基础的 API 调用。

---

### 3. 问题驱动 — 缺什么学什么

> 放弃「从头看到尾」的被动学习；用实战需求倒逼知识点，缺什么学什么，遇到坑再补理论

**核心面试问题驱动学习**：
- MVVM 和 MVI 核心区别？
- 组件化模块间通信方案？
- ARouter 路由表如何生成？
- Clean Architecture 三层职责？
- Tinker Dex 差量替换原理？

**行动**：选一个问题 → 尝试自己回答 → 对比标准答案 → 补知识点 → 再回答一遍。

---

### 4. 单点击穿 — 一次只攻克一个

> 把复杂大系统拆成独立小知识点，一次只攻克一个，学完立刻落地使用；不并行啃多个难点

**建议学习顺序**（一次只学一个子模块）：
1. MVVM（ViewModel+Repository+DataBinding）
2. MVI（State/Intent/Action+单向数据流）
3. Clean Architecture（Domain/Data/Presentation+UseCase+DIP）
4. 组件化（ARouter+接口下沉+Gradle模块管理）
5. 插件化（Tinker Dex差量+Shadow ClassLoader Hook）

**规则**：当前子模块没跑通最小闭环前，绝不开下一个。

---

### 5. 学以致用 — 学了立刻用

> 学到的每一个知识点，必须立刻动手复用一次；不囤积知识，只留存能落地、能复现的内容

**每个子模块的实战任务**：
- 对比 MVVM 和 MVI 同功能实现
- 单模块拆分为 3 模块+壳工程
- ARouter 拦截器（登录校验）
- Tinker 热修复 Demo

---

### 6. 先会后懂 — 允许先会用

> 先用起来，再慢慢吃透底层；不要卡在一个细节死磕，耽误整体进度

**先用起来的关键 API**（暂时不需要深究源码）：
- `@HiltViewModel`/`@Composable(state,onIntent)`/`@Route(path)`/`interface IAccountService`

---

### 7. 模板沉淀 — 把知识变成资产

> 把常用写法、固定架构、避坑方案，沉淀成私有模板/笔记/代码片段；从「临时记忆」变成「长期能力」

**建议沉淀的模板**：
- MVVM 项目结构模板 / MVI State/Intent/Reducer 模板 / 组件化模块划分模板 / ARouter 路由配置模板

---

### 8. 新旧嫁接 — 用已知类比未知

> 用已会的技能类比映射新技能，找异同、记差异，不用从零陌生认知

**本模块的类比映射**：
- MVVM→观察者模式 / MVI→银行柜台 / Clean Architecture→三层办公楼 / 组件化→乐高积木

---

### 9. 刻意屏蔽 — 先学核心再补边缘

> 主动过滤冷门、边角、工作用不到的知识；只学核心主干、高频必备，边缘知识用到再查

**本模块的核心主干**（必须掌握）：
- MVVM vs MVI 数据流 / Repository 模式 / 组件化通信 4 方案 / ARouter APT 路由表 / Clean Architecture 分层

**本模块的边缘知识**（先跳过，用到再学）：
- 插件化 ClassLoader 双亲委派突破 / Tinker 资源补丁 / VirtualAPK 对比

---

### 10. 闭环复盘 — 串联成体系

> 定期复盘：把零散知识点串联成逻辑链路，从「会零散 API」升级为「懂架构、懂设计、能独立搭建」

**复盘检查清单**：
- [ ] 画出 MVVM 和 MVI 数据流向图
- [ ] 画出 Clean Architecture 三层依赖图
- [ ] 解释组件化 vs 插件化区别
- [ ] 设计中大型 App 架构方案

**频率**：每完成一个阶段的 3 个子模块后，做一次串联复盘。
