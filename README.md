## 📖 使用指南

每个模块包含两个核心学习文件：

| 文件 | 说明 | 用途 |
|------|------|------|
| `知识骨架.xmind` | XMind 思维导图 | 建立全局认知地图，用 XMind/Zen 打开 |
| `学习指南.md` | 10 条原则落地方案 | 按阶段学习，最小闭环 + 面试题 + 避坑 |

**推荐学习流程**：
1. 打开 `知识骨架.xmind`，花 5 分钟浏览全局
2. 阅读 `学习指南.md`，找到最小闭环项目
3. 按阶段逐点攻克，一次只学一个子模块
4. 学完立刻动手实践，沉淀代码模板
5. 每 3 个子模块复盘串联

## 🐍 10 条学习原则速查

| # | 原则 | 一句话 | 应用 |
|:--:|------|--------|------|
| 1 | 骨架先行 | 先有地图，再逛景点 | 每个模块先看 XMind 全景 |
| 2 | 最小闭环 | 先跑通完整流程 | 每个模块有最小闭环项目 |
| 3 | 问题驱动 | 缺什么学什么 | 以面试题为学习入口 |
| 4 | 单点击穿 | 一次只攻一个点 | 按学习指南的阶段顺序 |
| 5 | 学以致用 | 学了立刻用 | 每个子模块有实战任务 |
| 6 | 先会后懂 | 先用起来，后懂原理 | 先背 API 用法，面试前补源码 |
| 7 | 模板沉淀 | 知识变成可复用资产 | 建议沉淀的模板清单 |
| 8 | 新旧嫁接 | 用已知类比未知 | Java→Kotlin 对照表 |
| 9 | 刻意屏蔽 | 先核心后边缘 | 核心主干 vs 边缘知识 |
| 10 | 闭环复盘 | 串联成体系 | 每 3 个子模块复盘 |
