# 08 性能优化

## 概览
安卓应用全链路性能优化，是 **30k+ 岗位最核心的区分维度**。面试官通过性能优化问题，同时考察：
- 基础功底（是否理解安卓渲染/内存/启动机制）
- 工具能力（能否用 Systrace/Perfetto/Memory Profiler 定位问题）
- 工程思维（是否有量化指标、优化闭环、防劣化机制）

## 面试核心考察模型

```
面试官提问路径:
  "你做过什么性能优化？"  
    → 候选人说做了什么
    → 追问: "为什么慢？你怎么定位的？"（工具能力）
    → 追问: "优化效果怎么量化的？"（工程思维）
    → 追问: "有没有引入新问题？怎么防止劣化？"（闭环能力）
    → 终极追问: "从系统层面(AMS/WMS/Zygote)解释一下原理"（技术深度）
```

## 性能优化黄金法则

1. **先测量再优化**：无量化不优化，基线数据是优化的起点
2. **二八原则**：80%的问题来自20%的代码，用工具定位热点
3. **空间换时间**：缓存、预加载、对象池
4. **善用异步**：主线程只做UI操作，IO/计算放子线程
5. **防劣化先行**：核心指标上CI门禁，每次提测自动跑性能基线
