## 🗺️ 知识骨架全景图

打开本目录下的 `知识骨架.xmind` 查看完整知识结构。

├── 📁 🐍 核心心法
    ├── 先有量化指标，后优化（数据驱动，不能盲猜）
    ├── 启动优化最容易出成绩，优先攻克
    ├── 内存/卡顿用 Systrace/Perfetto 解决 80% 问题
    ├── 包体积 → 瘦身三板斧: 混淆/压缩/动态下发
├── 📁 📚 学习路径 (6阶段)
    ├── 📁 阶段1: 启动速度优化
        ├── 冷启动 4 阶段: fork→bindApp→Activity.onCreate→首帧
        ├── 启动框架: App Startup / 异步初始化 DAG
        ├── 懒加载: ViewStub/Stub 占位
        ├── Theme 优化: windowBackground 秒开白屏
        ├── 量化: adb shell am start -W 测量
    ├── 📁 阶段2: UI 卡顿优化
        ├── 16ms 渲染帧率 — VSync 信号驱动
        ├── 过度绘制检测 — GPU 呈现模式分析
        ├── 布局优化: merge/ViewStub/ConstraintLayout
        ├── RecyclerView 优化: setHasFixedSize/ViewPool/DiffUtil
    ├── 📁 阶段3: 内存泄漏优化
        ├── GC Root 可达性分析
        ├── 5 大泄漏场景: Handler/匿名内部类/静态变量/单例/资源
        ├── MAT 支配树 — 找 Retained Heap 最大对象
        ├── LeakCanary 2 自动化检测
    ├── 📁 阶段4: 功耗优化
        ├── Battery Historian v2 — 电量分析
        ├── WakeLock — 避免持锁不释放
        ├── JobScheduler/WorkManager — 统一调度
        ├── Doze/App Standby — 系统省电模式适配
    ├── 📁 阶段5: 包体积优化
        ├── AndResGuard — 资源混淆（微信方案）
        ├── ProGuard/R8 — 代码混淆三阶段
        ├── 动态下发 — 动态 so/资源按需下载
        ├── WebP/SVG — 替换 PNG 图片资源
    ├── 📁 阶段6: 网络优化
        ├── HTTP/2 — 多路复用 + 头部压缩
        ├── 连接池 — OkHttp ConnectionPool(5,5)
        ├── HTTPDNS — 域名解析防劫持
        ├── 弱网优化 — QUIC / 预加载
├── 📁 🎯 面试重点
    ├── 冷启动完整阶段及每阶段优化手段
    ├── Systrace 中掉帧判断（Frame 圆圈颜色）
    ├── 内存泄漏的 3 种定位方法及对比
    ├── R8 三阶段做了什么
    ├── 网络请求耗时拆解 (DNS→TCP→TLS→Request→Response)
├── 📁 ⚡ 最小闭环项目
    ├── 拿一个真实项目 → Systrace 抓取 → 找出 3 个优化点
    ├── 量化优化效果 → 形成优化报告
├── 📁 🔧 避坑指南
    ├── 不要过早优化 — 先 profiling 再优化
    ├── 启动优化误区: 全部异步 → 依赖错乱
    ├── LeakCanary 线上不要开（性能开销大）
    ├── WebP 兼容性 — Android 4.0+ 原生支持

## 🎯 10 条学习原则落地指南

### 1. 骨架先行 — 建立整体认知地图

> 先搞懂整体结构、模块划分、核心链路，不一开始扎进零散语法/API；先有地图，再逛景点

**本模块骨架**：启动速度→UI卡顿→内存泄漏→功耗→包体积→网络优化，全链路覆盖

**行动**：打开 `知识骨架.xmind`，花 5 分钟浏览全部节点，建立模块全局印象，再按阶段逐一攻克。

---

### 2. 最小闭环 — 先跑通再深挖

> 先做一个能完整运行的最小可用版本，不求完美、不求精通；先建立体感和全局流程，再回头拆解底层原理

**本模块最小闭环项目**：
- **项目**：对现有项目全链路性能诊断
- **工具**：Systrace/Perfetto+Memory Profiler+CPU Profiler
- **闭环目标**：找出 3 个优化点→量化效果→形成报告

**检查点**：能否独立跑通这个闭环？如果能，说明框架理解正确；如果不能，回头补最基础的 API 调用。

---

### 3. 问题驱动 — 缺什么学什么

> 放弃「从头看到尾」的被动学习；用实战需求倒逼知识点，缺什么学什么，遇到坑再补理论

**核心面试问题驱动学习**：
- 冷启动分哪几个阶段？
- Systrace 中如何判断掉帧？
- 内存泄漏 3 种定位方法？
- R8 三阶段各做了什么？
- 网络请求耗时拆解？

**行动**：选一个问题 → 尝试自己回答 → 对比标准答案 → 补知识点 → 再回答一遍。

---

### 4. 单点击穿 — 一次只攻克一个

> 把复杂大系统拆成独立小知识点，一次只攻克一个，学完立刻落地使用；不并行啃多个难点

**建议学习顺序**（一次只学一个子模块）：
1. 启动速度（冷启动4阶段+StartUp+懒加载）
2. UI卡顿（16ms渲染+过度绘制+RecyclerView优化）
3. 内存泄漏（GC Root+5大泄漏场景+MAT+LeakCanary）
4. 功耗（Battery Historian+WakeLock+WorkManager）
5. 包体积（AndResGuard+R8+动态下发+WebP）
6. 网络优化（HTTP/2+连接池+HTTPDNS+QUIC）

**规则**：当前子模块没跑通最小闭环前，绝不开下一个。

---

### 5. 学以致用 — 学了立刻用

> 学到的每一个知识点，必须立刻动手复用一次；不囤积知识，只留存能落地、能复现的内容

**每个子模块的实战任务**：
- Systrace/Perfetto 抓取启动 trace 分析
- LeakCanary 检出并修复 3 个泄漏
- APK Analyzer 分析包体积
- 对比 R8 开启前后 APK 差异

---

### 6. 先会后懂 — 允许先会用

> 先用起来，再慢慢吃透底层；不要卡在一个细节死磕，耽误整体进度

**先用起来的关键 API**（暂时不需要深究源码）：
- `adb shell am start -W` / Systrace/Perfetto / LeakCanary / APK Analyzer

---

### 7. 模板沉淀 — 把知识变成资产

> 把常用写法、固定架构、避坑方案，沉淀成私有模板/笔记/代码片段；从「临时记忆」变成「长期能力」

**建议沉淀的模板**：
- 性能优化检查清单 / Systrace 常见问题速查 / 内存泄漏 5 场景+方案 / 网络优化方案对比

---

### 8. 新旧嫁接 — 用已知类比未知

> 用已会的技能类比映射新技能，找异同、记差异，不用从零陌生认知

**本模块的类比映射**：
- 启动→汽车起步 / 内存泄漏→水槽漏水 / RecyclerView→流水线复用 / R8→文件压缩

---

### 9. 刻意屏蔽 — 先学核心再补边缘

> 主动过滤冷门、边角、工作用不到的知识；只学核心主干、高频必备，边缘知识用到再查

**本模块的核心主干**（必须掌握）：
- 冷启动 4 阶段与优化 / Systrace Frame 渲染分析 / GC Root 可达性 / R8 三阶段 / OkHttp 连接池

**本模块的边缘知识**（先跳过，用到再学）：
- Systrace 自定义 Trace 埋点 / Battery Historian 复杂分析 / QUIC 细节 / PGO

---

### 10. 闭环复盘 — 串联成体系

> 定期复盘：把零散知识点串联成逻辑链路，从「会零散 API」升级为「懂架构、懂设计、能独立搭建」

**复盘检查清单**：
- [ ] 独立完成 App 启动优化（量化效果）
- [ ] Systrace/Perfetto 定位卡顿
- [ ] 识别 5 种常见泄漏场景
- [ ] 说出包体积优化完整方案

**频率**：每完成一个阶段的 3 个子模块后，做一次串联复盘。
