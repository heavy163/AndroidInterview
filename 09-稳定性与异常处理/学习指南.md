## 🗺️ 知识骨架全景图

打开本目录下的 `知识骨架.xmind` 查看完整知识结构。

├── 📁 🐍 核心心法
    ├── Crash 率 < 0.1% 是硬指标，ANR 率 < 0.05%
    ├── 先建立监控（发现率），再做修复（解决率）
    ├── ANR → traces.txt 解读是关键技能
    ├── 崩溃 → 堆栈符号化 → 归因 → 闭环
├── 📁 📚 学习路径 (4阶段)
    ├── 📁 阶段1: ANR 分析与解决
        ├── 三种 ANR: Input(5s)/Service(20s/10s)/Broadcast(10s/60s)
        ├── traces.txt 解读 — 找主线程状态 (native/java/waiting)
        ├── 常见原因: 主线程 IO、死锁、Binder 阻塞
        ├── ANR-WatchDog — 开源的 ANR 监控方案
    ├── 📁 阶段2: 崩溃分析
        ├── Java Crash — UncaughtExceptionHandler 拦截
        ├── Native Crash — 信号处理器 + tombstone
        ├── OOM — Runtime.maxMemory 监控 + 内存泄漏链路
        ├── Crash 堆栈符号化 — ndk-stack / addr2line
    ├── 📁 阶段3: System Restart
        ├── system_server 重启 — WatchDog 触发
        ├── Kernel Panic — last_kmsg 分析
        ├── 重启信息收集 — 开机后读取 /sys/fs/pstore
    ├── 📁 阶段4: 异常监控体系
        ├── 自定义 UncaughtExceptionHandler 链
        ├── 全链路追踪 traceId — 从客户端到服务端
        ├── OOM 监控 — 定时采集 Runtime 内存 + 阈值告警
        ├── ANR 监控 — FileObserver 监听 /data/anr/
├── 📁 🎯 面试重点
    ├── ANR traces.txt 中主线程 'waiting to lock' vs 'native' 的区别
    ├── Java crash vs Native crash 捕获机制差异
    ├── OOM 预防：大图加载 + LRU 缓存 + 降采样
    ├── 线上异常监控架构设计
├── 📁 ⚡ 最小闭环项目
    ├── 搭建一个简单的异常上报系统
    ├── UncaughtExceptionHandler → 持久化 → 上传
├── 📁 🔧 避坑指南
    ├── UncaughtExceptionHandler 链 — 保留系统默认处理
    ├── Native Crash 信号处理不能做复杂操作
    ├── OOM 监控 — fork 子进程 dump hprof (KOOM 方案)

## 🎯 10 条学习原则落地指南

### 1. 骨架先行 — 建立整体认知地图

> 先搞懂整体结构、模块划分、核心链路，不一开始扎进零散语法/API；先有地图，再逛景点

**本模块骨架**：ANR 分析→崩溃捕获→SystemRestart→异常监控体系

**行动**：打开 `知识骨架.xmind`，花 5 分钟浏览全部节点，建立模块全局印象，再按阶段逐一攻克。

---

### 2. 最小闭环 — 先跑通再深挖

> 先做一个能完整运行的最小可用版本，不求完美、不求精通；先建立体感和全局流程，再回头拆解底层原理

**本模块最小闭环项目**：
- **项目**：搭建异常上报系统
- **闭环目标**：UncaughtExceptionHandler→持久化→上传→服务端聚合

**检查点**：能否独立跑通这个闭环？如果能，说明框架理解正确；如果不能，回头补最基础的 API 调用。

---

### 3. 问题驱动 — 缺什么学什么

> 放弃「从头看到尾」的被动学习；用实战需求倒逼知识点，缺什么学什么，遇到坑再补理论

**核心面试问题驱动学习**：
- ANR traces.txt 解读？
- Java vs Native crash 捕获差异？
- OOM 预防与监控？

**行动**：选一个问题 → 尝试自己回答 → 对比标准答案 → 补知识点 → 再回答一遍。

---

### 4. 单点击穿 — 一次只攻克一个

> 把复杂大系统拆成独立小知识点，一次只攻克一个，学完立刻落地使用；不并行啃多个难点

**建议学习顺序**（一次只学一个子模块）：
1. ANR 三类型超时+traces.txt 解读
2. Java/Native Crash 捕获原理
3. System Restart 分析
4. 线上异常监控架构

**规则**：当前子模块没跑通最小闭环前，绝不开下一个。

---

### 5. 学以致用 — 学了立刻用

> 学到的每一个知识点，必须立刻动手复用一次；不囤积知识，只留存能落地、能复现的内容

**每个子模块的实战任务**：
- 手动制造 ANR 并解读
- 自定义 UncaughtExceptionHandler
- 设计 crash 链路追踪

---

### 6. 先会后懂 — 允许先会用

> 先用起来，再慢慢吃透底层；不要卡在一个细节死磕，耽误整体进度

**先用起来的关键 API**（暂时不需要深究源码）：
- 核心：ANR 三类型+traces.txt / Java/Native Crash 捕获 / OOM 监控 / 全链路 traceId
