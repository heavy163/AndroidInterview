## 🎯 10 条学习原则落地指南

### 1. 骨架先行 — 建立整体认知地图

> 先搞懂整体结构、模块划分、核心链路，不一开始扎进零散语法/API；先有地图，再逛景点

**本模块骨架**：四大组件生命周期 → 组件通信 → Binder IPC → Handler 消息循环，从表象到底层

**行动**：打开 `知识骨架.xmind`，花 5 分钟浏览全部节点，建立模块全局印象，再按阶段逐一攻克。

---

### 2. 最小闭环 — 先跑通再深挖

> 先做一个能完整运行的最小可用版本，不求完美、不求精通；先建立体感和全局流程，再回头拆解底层原理

**本模块最小闭环项目**：
- **项目**：IPC 跨进程通信 Demo
- **技术栈**：AIDL + Service
- **闭环目标**：定义 AIDL 接口 → Service 实现 → 客户端调用

**检查点**：能否独立跑通这个闭环？如果能，说明框架理解正确；如果不能，回头补最基础的 API 调用。

---

### 3. 问题驱动 — 缺什么学什么

> 放弃「从头看到尾」的被动学习；用实战需求倒逼知识点，缺什么学什么，遇到坑再补理论

**核心面试问题驱动学习**：
- 点击图标到 Activity 展示的完整流程？
- Binder 为什么只拷贝一次？
- Handler 消息延迟原因？
- Activity 四种启动模式？
- Service 与 Thread 区别？

**行动**：选一个问题 → 尝试自己回答 → 对比标准答案 → 补知识点 → 再回答一遍。

---

### 4. 单点击穿 — 一次只攻克一个

> 把复杂大系统拆成独立小知识点，一次只攻克一个，学完立刻落地使用；不并行啃多个难点

**建议学习顺序**（一次只学一个子模块）：
1. 四大组件全景（生命周期/启动模式/注册方式）
2. 生命周期管理（Activity/Fragment/异常恢复/ViewModel）
3. 组件间通信（Intent→EventBus→LiveEventBus→ContentProvider）
4. Binder 机制深度（mmap→AIDL 生成类→ashmem）
5. Handler 消息机制源码级（MessageQueue→Looper→同步屏障→IdleHandler）

**规则**：当前子模块没跑通最小闭环前，绝不开下一个。

---

### 5. 学以致用 — 学了立刻用

> 学到的每一个知识点，必须立刻动手复用一次；不囤积知识，只留存能落地、能复现的内容

**每个子模块的实战任务**：
- 实现跨进程 Service 调用
- 写 Handler 消息机制 Demo（含同步屏障+IdleHandler）
- 四种启动模式效果演示

---

### 6. 先会后懂 — 允许先会用

> 先用起来，再慢慢吃透底层；不要卡在一个细节死磕，耽误整体进度

**先用起来的关键 API**（暂时不需要深究源码）：
- `startActivity`/`bindService`/`registerReceiver`/`Handler.post`

---

### 7. 模板沉淀 — 把知识变成资产

> 把常用写法、固定架构、避坑方案，沉淀成私有模板/笔记/代码片段；从「临时记忆」变成「长期能力」

**建议沉淀的模板**：
- Activity 生命周期模板 / AIDL 接口模板 / Handler 消息机制图解 / 四大组件对比速查表

---

### 8. 新旧嫁接 — 用已知类比未知

> 用已会的技能类比映射新技能，找异同、记差异，不用从零陌生认知

**本模块的类比映射**：
- Binder→Linux 管道/socket 对比 / Handler→EventLoop 类比 / ContentProvider→RESTful API

---

### 9. 刻意屏蔽 — 先学核心再补边缘

> 主动过滤冷门、边角、工作用不到的知识；只学核心主干、高频必备，边缘知识用到再查

**本模块的核心主干**（必须掌握）：
- Activity 四种启动模式 / Service 三种形态 / Handler 源码 / Binder 一次拷贝原理

**本模块的边缘知识**（先跳过，用到再学）：
- BroadcastReceiver 有序广播细节 / ContentProvider 批量操作 / Binder 线程池 16 上限

---

### 10. 闭环复盘 — 串联成体系

> 定期复盘：把零散知识点串联成逻辑链路，从「会零散 API」升级为「懂架构、懂设计、能独立搭建」

**复盘检查清单**：
- [ ] 画出 Activity 启动流程图
- [ ] 解释 Binder mmap 一次拷贝
- [ ] 手写简化版 Handler
- [ ] 说出四大组件使用场景和限制

**频率**：每完成一个阶段的 3 个子模块后，做一次串联复盘。
