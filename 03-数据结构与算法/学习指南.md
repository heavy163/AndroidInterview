## 🎯 10 条学习原则落地指南

### 1. 骨架先行 — 建立整体认知地图

> 先搞懂整体结构、模块划分、核心链路，不一开始扎进零散语法/API；先有地图，再逛景点

**本模块骨架**：常用数据结构 API → 排序/查找算法 → 动态规划 → 安卓中的算法落地

**行动**：打开 `知识骨架.xmind`，花 5 分钟浏览全部节点，建立模块全局印象，再按阶段逐一攻克。

---

### 2. 最小闭环 — 先跑通再深挖

> 先做一个能完整运行的最小可用版本，不求完美、不求精通；先建立体感和全局流程，再回头拆解底层原理

**本模块最小闭环项目**：
- **刷题路径**：数组(5题)→链表(5题)→二叉树(5题)→DP(5题)
- **闭环目标**：每个数据结构核心操作能手写，能口述复杂度

**检查点**：能否独立跑通这个闭环？如果能，说明框架理解正确；如果不能，回头补最基础的 API 调用。

---

### 3. 问题驱动 — 缺什么学什么

> 放弃「从头看到尾」的被动学习；用实战需求倒逼知识点，缺什么学什么，遇到坑再补理论

**核心面试问题驱动学习**：
- HashMap put 完整流程？
- SparseArray 和 HashMap 本质区别？
- 手写快排+复杂度分析？
- LRU Cache 实现？

**行动**：选一个问题 → 尝试自己回答 → 对比标准答案 → 补知识点 → 再回答一遍。

---

### 4. 单点击穿 — 一次只攻克一个

> 把复杂大系统拆成独立小知识点，一次只攻克一个，学完立刻落地使用；不并行啃多个难点

**建议学习顺序**（一次只学一个子模块）：
1. 数据结构基础（ArrayList扩容/HashMap源码/SparseArray/ArrayMap）
2. 排序算法手写（快排/归并/堆排）+复杂度推导
3. 查找与遍历（二分/BFS/DFS/布隆过滤器）
4. 动态规划（0-1背包/LCS）

**规则**：当前子模块没跑通最小闭环前，绝不开下一个。

---

### 5. 学以致用 — 学了立刻用

> 学到的每一个知识点，必须立刻动手复用一次；不囤积知识，只留存能落地、能复现的内容

**每个子模块的实战任务**：
- 手写 HashMap 简化版
- 手写 LRU Cache
- 手写快排+三路分区优化

---

### 6. 先会后懂 — 允许先会用

> 先用起来，再慢慢吃透底层；不要卡在一个细节死磕，耽误整体进度

**先用起来的关键 API**（暂时不需要深究源码）：
- `HashMap.put/get` / `SparseArray.put/get` / `ArrayDeque` / `Collections.sort()`

---

### 7. 模板沉淀 — 把知识变成资产

> 把常用写法、固定架构、避坑方案，沉淀成私有模板/笔记/代码片段；从「临时记忆」变成「长期能力」

**建议沉淀的模板**：
- 快排/归并/堆排手写模板 / BFS/DFS 框架模板 / LRU Cache 模板 / 数据复杂度速查表

---

### 8. 新旧嫁接 — 用已知类比未知

> 用已会的技能类比映射新技能，找异同、记差异，不用从零陌生认知

**本模块的类比映射**：
- HashMap→图书馆索引 / SparseArray→字典拼音 / BFS→水波 / DFS→走迷宫

---

### 9. 刻意屏蔽 — 先学核心再补边缘

> 主动过滤冷门、边角、工作用不到的知识；只学核心主干、高频必备，边缘知识用到再查

**本模块的核心主干**（必须掌握）：
- HashMap 源码（put/get/resize/树化）/ ArrayList 扩容 / 快排+归并 / 二分查找 / SparseArray

**本模块的边缘知识**（先跳过，用到再学）：
- 红黑树自平衡细节 / 布隆过滤器推导 / DP 状态压缩 / 图高级算法

---

### 10. 闭环复盘 — 串联成体系

> 定期复盘：把零散知识点串联成逻辑链路，从「会零散 API」升级为「懂架构、懂设计、能独立搭建」

**复盘检查清单**：
- [ ] 手写 HashMap 简化版
- [ ] 手写快排和归并
- [ ] 解释 SparseArray 省内存原因
- [ ] LeetCode Hot100 完成30+题

**频率**：每完成一个阶段的 3 个子模块后，做一次串联复盘。
