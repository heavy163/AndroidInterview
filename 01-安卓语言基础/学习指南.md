## 🎯 10 条学习原则落地指南

### 1. 骨架先行 — 建立整体认知地图

> 先搞懂整体结构、模块划分、核心链路，不一开始扎进零散语法/API；先有地图，再逛景点

**本模块骨架**：Kotlin 核心特性 → 协程并发 → Java 进阶 → KMM 跨平台，四层递进

**行动**：打开 `知识骨架.xmind`，花 5 分钟浏览全部节点，建立模块全局印象，再按阶段逐一攻克。

---

### 2. 最小闭环 — 先跑通再深挖

> 先做一个能完整运行的最小可用版本，不求完美、不求精通；先建立体感和全局流程，再回头拆解底层原理

**本模块最小闭环项目**：
- **项目**：天气预报 App
- **技术栈**：Kotlin + MVVM + 协程 + Flow + Retrofit
- **闭环目标**：搜索城市 → 网络请求 → UI 更新

**检查点**：能否独立跑通这个闭环？如果能，说明框架理解正确；如果不能，回头补最基础的 API 调用。

---

### 3. 问题驱动 — 缺什么学什么

> 放弃「从头看到尾」的被动学习；用实战需求倒逼知识点，缺什么学什么，遇到坑再补理论

**核心面试问题驱动学习**：
- suspend 函数底层实现？
- Flow vs LiveData vs StateFlow 选用场景？
- 协程取消的协作机制？
- Kotlin inline 如何减少对象创建？

**行动**：选一个问题 → 尝试自己回答 → 对比标准答案 → 补知识点 → 再回答一遍。

---

### 4. 单点击穿 — 一次只攻克一个

> 把复杂大系统拆成独立小知识点，一次只攻克一个，学完立刻落地使用；不并行啃多个难点

**建议学习顺序**（一次只学一个子模块）：
1. Kotlin 核心特性（扩展函数/高阶函数/委托/密封类/数据类）
2. Kotlin 协程（suspend→Flow→StateFlow→结构化并发）
3. Java 进阶（JVM 内存模型/GC/APT）
4. KMM 跨平台

**规则**：当前子模块没跑通最小闭环前，绝不开下一个。

---

### 5. 学以致用 — 学了立刻用

> 学到的每一个知识点，必须立刻动手复用一次；不囤积知识，只留存能落地、能复现的内容

**每个子模块的实战任务**：
- 用扩展函数封装 View 通用操作
- 用 sealed class 定义网络请求状态机
- 用 Flow 实现搜索防抖

---

### 6. 先会后懂 — 允许先会用

> 先用起来，再慢慢吃透底层；不要卡在一个细节死磕，耽误整体进度

**先用起来的关键 API**（暂时不需要深究源码）：
- `suspend fun` / `viewModelScope.launch { }` / `flow { emit(value) }` / `MutableStateFlow(initial)`

---

### 7. 模板沉淀 — 把知识变成资产

> 把常用写法、固定架构、避坑方案，沉淀成私有模板/笔记/代码片段；从「临时记忆」变成「长期能力」

**建议沉淀的模板**：
- 协程网络请求模板 / sealed class 状态机模板 / Flow 搜索防抖模板

---

### 8. 新旧嫁接 — 用已知类比未知

> 用已会的技能类比映射新技能，找异同、记差异，不用从零陌生认知

**本模块的类比映射**：
- Java 匿名内部类→Kotlin lambda / Java Future→suspend / Java RxJava→Flow

---

### 9. 刻意屏蔽 — 先学核心再补边缘

> 主动过滤冷门、边角、工作用不到的知识；只学核心主干、高频必备，边缘知识用到再查

**本模块的核心主干**（必须掌握）：
- Kotlin 扩展函数/高阶函数/委托/密封类/数据类 / 协程基础 / Flow/StateFlow / JVM 堆栈结构/GC Roots

**本模块的边缘知识**（先跳过，用到再学）：
- KMM expect/actual 详细用法 / inline/noinline 编译细节 / Kotlin 编译器插件开发

---

### 10. 闭环复盘 — 串联成体系

> 定期复盘：把零散知识点串联成逻辑链路，从「会零散 API」升级为「懂架构、懂设计、能独立搭建」

**复盘检查清单**：
- [ ] 独立用 Kotlin+协程写网络请求页面
- [ ] 解释 suspend 函数 CPS 转换
- [ ] 说清 Flow vs LiveData vs StateFlow 选择标准
- [ ] 画出 JVM 内存区域分布图

**频率**：每完成一个阶段的 3 个子模块后，做一次串联复盘。
