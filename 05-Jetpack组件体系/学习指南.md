## 🗺️ 知识骨架全景图

打开本目录下的 `知识骨架.xmind` 查看完整知识结构。

├── 📁 🐍 核心心法
    ├── 先跑通一个 Compose + MVVM 的完整页面（闭环）
    ├── ViewModel/LiveData → 状态管理基石，先掌握
    ├── Room → 声明式数据库，背下三剑客注解

    ├── Compose → 声明式 UI 是未来，先学再用 View

        ├── ViewModel — 生命周期感知 + SavedStateHandle
        ├── LiveData — 粘性事件 + observe/livedata-ktx
        ├── MediatorLiveData — 多源合并
        ├── StateFlow vs LiveData — 选用策略
    ├── 📁 阶段2: Room 数据库
        ├── @Entity/@DAO/@Database 三剑客
        ├── Migration — 手动迁移 + 自动迁移(2.4+)
        ├── TypeConverter — 复杂类型转换
        ├── Flow 响应式查询 — 数据变更自动通知
    ├── 📁 阶段3: Navigation
        ├── NavGraph — 声明式导航图
        ├── SafeArgs — 类型安全传参
        ├── DeepLink — URI/PendingIntent 跳转
        ├── 多返回栈 — Android 12+ support
    ├── 📁 阶段4: Hilt 依赖注入
        ├── @HiltAndroidApp → @AndroidEntryPoint → @Inject
        ├── Scope 层级: Singleton > ViewModel > Fragment
        ├── 与 Dagger 对比 — 模板代码减少 80%
        ├── 自定义 Scope + @Binds/@Provides
    ├── 📁 阶段5: Compose 声明式 UI
        ├── 重组 Recomposition — 核心机制
        ├── remember/mutableStateOf — 状态管理
        ├── Modifier 链 — 样式/行为/布局
        ├── 副作用: LaunchedEffect/DisposableEffect/SideEffect
    ├── 📁 阶段6: 其他组件
        ├── WorkManager — 约束条件调度
        ├── DataStore — Proto/Preferences 替换 SP
        ├── Paging 3 — 分页加载
├── 📁 🎯 面试重点
    ├── ViewModel 如何在横竖屏切换时存活
    ├── Compose 重组 vs View invalidate 区别
    ├── Hilt 与 Dagger 核心区别
    ├── Room Flow 查询如何感知数据变更
├── 📁 ⚡ 最小闭环项目
    ├── 项目: 记事本 App (Compose + Room + Hilt + MVVM)
    ├── 增删改查 + Flow 响应式列表
├── 📁 🔧 避坑指南
    ├── LiveData setValue/postValue 线程限制
    ├── Room 主线程查询 StrictMode 检测
    ├── Compose 状态提升 — 避免状态下沉
    ├── Hilt Fragment 必须用 @AndroidEntryPoint

## 🎯 10 条学习原则落地指南

### 1. 骨架先行 — 建立整体认知地图

> 先搞懂整体结构、模块划分、核心链路，不一开始扎进零散语法/API；先有地图，再逛景点

**本模块骨架**：ViewModel/LiveData→Room→Navigation→Hilt→Compose，6 组件递进

**行动**：打开 `知识骨架.xmind`，花 5 分钟浏览全部节点，建立模块全局印象，再按阶段逐一攻克。

---

### 2. 最小闭环 — 先跑通再深挖

> 先做一个能完整运行的最小可用版本，不求完美、不求精通；先建立体感和全局流程，再回头拆解底层原理

**本模块最小闭环项目**：
- **项目**：记事本 App
- **技术栈**：Compose + Room + Hilt + MVVM
- **闭环目标**：笔记增删改查 + Flow 响应式列表

**检查点**：能否独立跑通这个闭环？如果能，说明框架理解正确；如果不能，回头补最基础的 API 调用。

---

### 3. 问题驱动 — 缺什么学什么

> 放弃「从头看到尾」的被动学习；用实战需求倒逼知识点，缺什么学什么，遇到坑再补理论

**核心面试问题驱动学习**：
- ViewModel 横竖屏切换如何保持数据？
- Compose 重组与 View invalidate 区别？
- Room Flow 查询如何感知变更？
- Hilt Scope 层级？
- StateFlow vs LiveData？

**行动**：选一个问题 → 尝试自己回答 → 对比标准答案 → 补知识点 → 再回答一遍。

---

### 4. 单点击穿 — 一次只攻克一个

> 把复杂大系统拆成独立小知识点，一次只攻克一个，学完立刻落地使用；不并行啃多个难点

**建议学习顺序**（一次只学一个子模块）：
1. ViewModel+LiveData（生命周期感知+SavedStateHandle）
2. Room（三剑客注解+Migration+Flow查询）
3. Navigation（NavGraph+SafeArgs+DeepLink）
4. Hilt（Scope层级+@Binds/@Provides）
5. Compose（重组机制+状态管理+副作用API）
6. 其他组件（WorkManager+DataStore+Paging3）

**规则**：当前子模块没跑通最小闭环前，绝不开下一个。

---

### 5. 学以致用 — 学了立刻用

> 学到的每一个知识点，必须立刻动手复用一次；不囤积知识，只留存能落地、能复现的内容

**每个子模块的实战任务**：
- ViewModel+StateFlow 改造 LiveData 项目
- Room Migration 测试
- DeepLink 从通知跳转详情页
- Compose 搜索列表页

---

### 6. 先会后懂 — 允许先会用

> 先用起来，再慢慢吃透底层；不要卡在一个细节死磕，耽误整体进度

**先用起来的关键 API**（暂时不需要深究源码）：
- `ViewModelProvider.get`/`liveData.observe`/`@Entity/@Composable`

---

### 7. 模板沉淀 — 把知识变成资产

> 把常用写法、固定架构、避坑方案，沉淀成私有模板/笔记/代码片段；从「临时记忆」变成「长期能力」

**建议沉淀的模板**：
- ViewModel+StateFlow 模板 / Room DAO CRUD 模板 / Hilt Module 模板 / Compose 状态管理模板

---

### 8. 新旧嫁接 — 用已知类比未知

> 用已会的技能类比映射新技能，找异同、记差异，不用从零陌生认知

**本模块的类比映射**：
- ViewModel→乐队指挥 / Room→Excel / Hilt→外卖平台 / Compose→公式推导

---

### 9. 刻意屏蔽 — 先学核心再补边缘

> 主动过滤冷门、边角、工作用不到的知识；只学核心主干、高频必备，边缘知识用到再查

**本模块的核心主干**（必须掌握）：
- ViewModel 生命周期+SavedStateHandle / Room 三剑客+Migration / Hilt Scope层级 / Compose 重组+remember+mutableStateOf+副作用

**本模块的边缘知识**（先跳过，用到再学）：
- Compose 与 View 互操作细节 / Room FTS4 / Paging3 RemoteMediator

---

### 10. 闭环复盘 — 串联成体系

> 定期复盘：把零散知识点串联成逻辑链路，从「会零散 API」升级为「懂架构、懂设计、能独立搭建」

**复盘检查清单**：
- [ ] Compose+Room+Hilt 写 CRUD 页面
- [ ] 解释 Compose 重组和智能跳过
- [ ] 画出 Hilt Scope 层级图
- [ ] 说出 ViewModel 和 onSaveInstanceState 互补关系

**频率**：每完成一个阶段的 3 个子模块后，做一次串联复盘。
