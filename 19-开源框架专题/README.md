## 面试考察模式

```
面试官: "你用过 Glide 吗？说说它的缓存机制"
      ↓ 
标准回答: 活动缓存(弱引用) → 内存缓存(LRU) → 磁盘缓存 → 网络源
      ↓
追问(体现深度): "活动缓存为什么用弱引用而不是软引用？"
      ↓
终极追问(区分30k+): "Glide如何实现Bitmap复用？说说从API 19到26的变化"
```

## 学习路径建议

1. **先攻核心三板斧**：Glide → OkHttp → Retrofit（网络+图片 = 80% 场景）
2. **再补路由与响应式**：ARouter → RxJava（架构必备）
3. **最后扩展专项**：EventBus / Tinker / MMKV / LeakCanary（视项目经历选择）
